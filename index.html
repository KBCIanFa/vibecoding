01 — The Concept

What is Vibe Coding?

Vibe Coding is a creative approach to building software where you describe what you want using natural language — your ideas, your personality, your vision — and let an AI like Claude write all the code.

💬

You describe, Claude builds

Write prompts in plain English. Describe the look, the feel, the function. No syntax to memorise, no stack to learn.

🔄

Iterate in real-time

Hate the colour? Say so. Want it bigger? Just ask. The loop between idea and result is measured in seconds.

🎨

Focus on creativity

Your energy goes into design thinking and storytelling — not debugging semicolons or reading Stack Overflow.

🚀

Ship something real

By the end of this tutorial you'll have a working, shareable HTML webpage. Not a mockup. An actual website.

The gatekeeping is gone

For decades, building software required years of training. You either learned to code, hired someone who had, or paid for expensive licensed tools. That cost — in time, money, and dependency on others — kept most people's ideas locked in their heads. Vibe Coding removes the gate entirely. If you can describe what you want, you can build it. Today.

🏗️

Enterprise tools are being disrupted

Platforms like Katalon, TestRail, and legacy test automation suites built their entire business on the idea that QA automation is hard. It required specialist engineers, expensive training, and long onboarding cycles. When a non-developer can describe a test scenario in plain English and have it running in minutes, the justification for those platforms collapses. The same disruption is hitting project management tools, BI dashboards, internal workflow builders — anything where the barrier was technical complexity.

🧑‍💼

Non-developers are becoming builders

Marketers are building their own campaign dashboards. Operations managers are creating internal tools without filing IT tickets. Designers are shipping their own prototypes directly to production. The line between "person with an idea" and "person who built the thing" has collapsed. This isn't developers being replaced — it's everyone else being unlocked.

🔁

The feedback loop is now seconds

Traditional development cycles: brief → spec → design → dev → QA → revisions → launch. Weeks or months. With Vibe Coding the cycle is: describe → see it → tweak → done. When iteration is free and instant, you stop planning and start experimenting. That changes how products get built at a fundamental level.

📉

The cost of a custom tool is approaching zero

A custom internal tool used to cost tens of thousands of dollars and months of engineering time. Now it costs an afternoon and a Claude subscription. Small businesses, solo operators, freelancers, and community organisations can all afford bespoke software. The economic case for buying bloated SaaS platforms instead of building exactly what you need is getting harder to justify.

🌏

It's especially powerful in Australia

Australia's small business economy is built on resourceful people doing more with less. We've always punched above our weight — and now Vibe Coding gives Australian SMEs, startups, and sole traders the same software-building capability that was once reserved for funded tech companies. Build for your specific market, your specific workflow, your specific clients — without a dev team.

"The next billion-dollar company might be built by someone who never wrote a single line of code — just someone who knew exactly what they wanted to build."

04 — The Tutorial

Build your first website
in 6 steps.

Each step teaches you a key Vibe Coding skill. Follow along in a new Claude chat as you read.

01

Open Claude and set the scene

Start a new conversation on claude.ai. Before giving Claude any creative instructions, establish what you're building. This first prompt is your "brief" — the more context you give, the better Claude performs.

Try this prompt

I want to build a personal portfolio website as a single self-contained HTML file. I'm a graphic designer based in Melbourne. My style is bold, modern, and a little quirky. I want visitors to immediately understand who I am and what I do. Let's build this together step by step — start by asking me anything you need to know.

💡 Asking Claude to ask you questions first produces dramatically better output. It forces Claude to gather your actual preferences rather than making assumptions.
02

Describe the look and feel

Vibe Coding is at its best when you lean into mood, texture, and emotion — not just specs. Use references, comparisons, and adjectives you'd normally use in a design brief.

Try this prompt

For the visual style: think dark background with a single electric accent colour — maybe neon green or hot pink. Big bold typography that feels like a magazine editorial. Clean sections with lots of breathing room. Subtle animations on scroll. The overall vibe should feel like a high-end creative studio, not a template.

WHAT CLAUDE WILL PRODUCE 
:root {
  --bg: #0a0a0a;
  --accent: #00ff88; /* your electric accent */
  --text: #f0f0f0;
}

h1 {
  font-size: clamp(3rem, 8vw, 7rem);
  font-weight: 800;
  letter-spacing: -0.03em;
}
03

Request the first working version

Now ask Claude to generate the actual code. Be specific about the format — a single HTML file is perfect for beginners because you can open it in any browser instantly, no server required.

Try this prompt

Please build the homepage now as a complete, self-contained HTML file. Include all CSS and JavaScript inside the same file — no external dependencies except Google Fonts. Include these sections: a hero with my name and tagline, a short about section, a 3-item portfolio grid (use placeholder images for now), and a contact section with my email.

💡 Saying "self-contained" is key. Claude will bundle everything into one file you can open locally — no npm, no server, no terminal needed.
04

Preview it & give feedback

Copy the code Claude gave you. Open a text editor (even Notepad works), paste the code, save as index.html, and open the file in your browser. Then come back to Claude with honest feedback — specific beats vague.

Practice: giving good feedback

Interactive

Compare vague vs specific feedback. Click the chips below to see the difference.

Vague feedback

Make it look better. The colours are off and it doesn't feel right.

❌ Claude has nothing to work with here. "Better" and "feel right" mean different things to everyone. This will produce a guess, not the outcome you want.
Specific feedback

The hero section is great but the portfolio grid feels too cramped — increase the gap between cards to 40px and give each card a slight drop shadow. Also the font in the about section is too small on mobile — bump it to at least 18px. Keep everything else the same.

✓ This tells Claude exactly what to change, what value to use, and what to preserve. Claude will nail this on the first try.
05

Add interactivity & polish

Once the layout is right, ask Claude to layer in animations, hover states, and micro-interactions. This is where Vibe Coding really shines — effects that would take hours to hand-code come in seconds.

Try this prompt

Now add some life to the page: 1) Fade-in animations for each section as the user scrolls down using the Intersection Observer API. 2) On the portfolio cards, add a smooth colour overlay on hover that reveals a "View Project" label. 3) Add a subtle parallax effect to the hero background. Keep all animations tasteful — nothing jarring.

CLAUDE WILL WRITE THIS FOR YOU 
// Intersection Observer for scroll animations
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
    }
  });
}, { threshold: 0.15 });

document.querySelectorAll('.section')
  .forEach(el => observer.observe(el));
06

Make it yours & share it

Replace all placeholder content with your real info. Then share it — upload to GitHub Pages (free), Netlify Drop, or even email the HTML file directly. You built a real website.

Try this prompt

Here's my real content to replace the placeholders — [paste your actual name, bio, project descriptions, and contact details]. Also make sure the page has good meta tags for SEO and social sharing. Finally, add a small "Built with Claude" credit in the footer — I'm proud of how this was made.

🚀 To deploy for free: go to netlify.com/drop, drag your HTML file onto the page, and your website is live on the internet in under 10 seconds. No account required.
05 — Pro Tips

Vibe Coding
best practices.

These habits separate a frustrating Vibe Coding session from a flow-state one.

TIP 01

One change at a time

Ask for one specific change per message. Bundling 10 requests often produces a broken mess. Small, precise prompts compound into great results.

TIP 02

Name your elements

Tell Claude what to call sections: "the .hero div", "the nav bar". Consistent naming lets you reference parts precisely in later prompts.

TIP 03

Use reference URLs

Say "I love the typography on stripe.com" or "the layout of linear.app". Claude understands design references and can draw from them.

TIP 04

Say what to preserve

When asking for changes, always say "keep everything else the same." This prevents Claude from helpfully 'improving' parts you already love.

TIP 05

Paste the current code

If you've modified the file yourself or the conversation is long, paste your current code when asking for updates. It ensures Claude works from reality, not memory.

TIP 06

Ask for explanations

After Claude writes something interesting, ask "what does that do?" You'll pick up concepts naturally — Vibe Coding is also a stealth way to learn real code.

07 — Power Up

Add AI to your website
with the Groq API.

Once your site is built, you can wire in a real AI brain. Groq offers a free API that lets your vibe-coded website talk to large language models at blazing speed — no backend required.

⚡

Extremely fast inference

Groq's hardware delivers some of the fastest LLM response times available — great for real-time chat and instant answers on your site.

🆓

Generous free tier

Get started for free at console.groq.com. No credit card needed for the free tier — ideal for experimenting and personal projects.

🔌

Works from plain HTML

The Groq API is a simple REST endpoint — Claude can write the fetch() call directly into your HTML file. No backend server needed.

01

Get your free API key

Head to console.groq.com/keys and sign in with Google or GitHub. Click "Create API Key", give it a name like my-vibe-site, and copy the key — it starts with gsk_...

🔒 Security note: For personal/demo projects an inline API key is fine. For anything public or production, move the key to a serverless function (Netlify Functions, Cloudflare Workers) so it isn't visible in your source code.
02

Tell Claude what you want to build

Describe the AI feature you want — a chat box, an answer bot, a content generator. Paste your API key into the prompt so Claude can wire it up immediately.

Try this prompt

Add an AI chat widget to my portfolio page. Use the Groq API (key: gsk_YOUR_KEY_HERE) with the model "llama3-8b-8192". The widget should appear as a floating button in the bottom-right corner. When clicked, it opens a small chat panel where visitors can ask me questions — answer as if you are me, a graphic designer in Melbourne. Keep the chat panel styled to match the existing dark aesthetic.

03

Understand the code Claude writes

Claude will insert a fetch() call to the Groq API. Here's what that looks like so you're not in the dark:

GROQ API CALL — WHAT CLAUDE GENERATES 
const response = await fetch(
  'https://api.groq.com/openai/v1/chat/completions',
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model: 'llama3-8b-8192',
      messages: [
        { role: 'system', content: systemPrompt },
        { role: 'user',   content: userMessage }
      ]
    })
  }
);

const data = await response.json();
const reply = data.choices[0].message.content;
💡 Ask Claude: "Explain what each part of this fetch call does in plain English." — this is a great way to learn real API concepts while Vibe Coding.
04

Choose the right model

Groq hosts several open-source models. Tell Claude which one to use based on what you need:

🦙

llama3-8b-8192

Fast, lightweight. Best for chat widgets, Q&A, and short responses. Ideal for most vibe-coded projects.

🔮

llama3-70b-8192

More powerful reasoning. Use when you need nuanced, longer, or more accurate responses.

💎

mixtral-8x7b-32768

Huge context window (32k tokens). Use when you need to process long documents or maintain extended conversations.
